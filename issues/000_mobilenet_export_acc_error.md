# MobileNetV2 量化精度调试总结

## 问题描述
在 MobileNetV2 模型部署过程中，发现量化模型（AXModel）与原始浮点模型（ONNX）的仿真结果存在较大数值差异：
- 最大相对差异：> 28x
- 余弦相似度：~0.995
- Top-k 分类匹配：完全正确

## 调试过程

### 1. 环境与配置检查
- **工具版本**: pulsar2 5.1-patch1 (commit aa417496)
- **目标芯片**: AX650 (NPU1模式)
- **原始模型**: torchvision.models.mobilenet_v2 (pretrained)
- **校准数据**: 32 个 COCO 图像样本，已进行 ImageNet 归一化

### 2. 配置问题排查
发现配置文件存在以下问题：
- **违规配置**: `"highest_mix_precision": true`（项目规范明确禁止）
- **归一化不一致**:
  - 校准数据已使用 ImageNet 归一化 (mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])
  - 但 `input_processors` 配置为 mean=[0,0,0], std=[1,1,1]
  - 量化配置表显示实际使用了 ImageNet 归一化参数

### 3. 精度对比实验
进行了多组对比实验：

| 实验 | 余弦相似度 | Top-1匹配 | 最大相对差异 | 关键发现 |
|------|------------|-----------|--------------|----------|
| 初始编译 | 0.115 | ❌ 不匹配 | 142.7 | 输入预处理不一致 |
| 双重归一化修复 | 0.115 | ❌ 不匹配 | 142.7 | 非主要问题 |
| KL量化方法 | 0.0925 | ❌ 不匹配 | 135.5 | 略有改善但不足 |
| **固定输入方案** | **0.9965** | ✅ 正确 | **30.72** | 关键突破 |
| **问题复现** | **0.9949** | ✅ 正确 | **28.81** | 确认可复现 |

### 4. 输出值深入分析
对最大相对差异进行根本原因分析：

```python
# 最大相对差异出现在索引 859
ONNX 值:     0.017592
AXModel 值:  0.524444
绝对差异:    0.506852
相对差异:    28.81x
```

**关键发现**:
1. 相对差异大的位置对应的 ONNX 输出值非常小（接近零）
2. 小的绝对误差（0.5）被小分母放大为大的相对差异
3. 输出值范围分析：
   - ONNX 输出范围: [-7.76, 12.46]
   - AXModel 输出范围: [-8.26, 12.46]
   - 均值绝对差异: 0.265
   - 余弦相似度: 0.995

## 根本原因

### 1. 评估指标选择不当
- **相对差异 (Relative Difference)** 不适用于输出包含接近零值的场景
- 分类模型的输出 logits 中许多类别得分接近零
- 小绝对误差导致极大的相对差异，产生误导

### 2. 配置违规影响
- `highest_mix_precision: true` 可能导致非最优量化策略
- 归一化参数不一致可能引入微小偏差

### 3. 量化误差特性
- 每层量化误差很小（余弦相似度 >0.98）
- 误差在多层网络中累积
- 累积误差在输出层表现为微小绝对差异

## 结论

### 1. 模型功能正确
- **余弦相似度 >0.99**：表明输出向量方向高度一致
- **Top-k 完全匹配**：分类结果与原始模型一致
- **绝对差异可控**：平均绝对差异 0.265，最大绝对差异 1.277

### 2. 相对差异指标误导
- 由于数学特性（小分母问题），相对差异不能准确反映模型精度
- 建议弃用相对差异作为主要评估指标

### 3. 推荐评估指标
对于分类模型的量化精度评估，建议使用：

| 指标 | 阈值 | 说明 |
|------|------|------|
| 余弦相似度 | >0.99 | 衡量输出向量方向一致性 |
| Top-1 匹配率 | 100% | 分类准确性 |
| Top-5 匹配率 | 100% | 分类鲁棒性 |
| 平均绝对差异 | <1.0 | 数值偏差程度 |
| 输出值相关性 | R² >0.98 | 线性相关性 |

## 建议

### 1. 配置修正
```json
{
  "input_processors": [{
    "mean": [0.485, 0.456, 0.406],
    "std": [0.229, 0.224, 0.225]
  }],
  // 移除 "highest_mix_precision": true
}
```

### 2. 评估策略调整
- 主要关注：余弦相似度、Top-k 匹配率
- 次要关注：绝对差异分布、输出值范围
- 避免使用：相对差异（特别是对于输出接近零的模型）

### 3. 后续步骤
1. **使用修正配置重新编译**：验证配置修正后的效果
2. **多样本测试**：使用更多测试样本验证稳定性
3. **板端验证**：在真实硬件上验证性能一致性
4. **文档更新**：将本次调试经验纳入项目最佳实践

## 技术收获

1. **预处理一致性至关重要**：输入数据的预处理必须与校准数据完全一致
2. **评估指标需谨慎选择**：不同指标反映不同维度的精度
3. **配置规范需严格遵守**：工具链的配置选项有明确的最佳实践
4. **量化误差分析需分层**：每层误差虽小，累积效应需关注
